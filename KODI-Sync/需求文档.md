# KODI Sync 功能需求文档

## 1. 概述

KODI Sync 是一个 Chataigne 自定义模块（UDP 类型），用于同步控制多台 KODI 播放器组成视频墙（Video Wall）。通过 UDP 广播实现帧级同步（<5ms 延迟）。

## 2. 架构

```
Chataigne (Mac)
    │
    ├─ KODI 模块 (WebSocket Client)
    │    └─ 控制单台 KODI（播放列表、音量、3D 等）
    │
    └─ KODI Sync 模块 (UDP type)
         └─ UDP 单播/IP → 所有 KODI 盒子（端口 9527）
              └─ service.chataigne-sync 插件
                   └─ xbmc.executeJSONRPC() 本地执行
```

### 通信协议

**发送（模块 → KODI）：**
```
CMD:VALUE\n
```

**反馈（KODI → 模块）：**
```
ACK:CMD\n
```

**支持的命令：**

| 命令 | 值 | 说明 |
|------|-----|------|
| PLAY | 无 | 播放 |
| PAUSE | 无 | 暂停 |
| STOP | 无 | 停止 |
| SEEK | 0-100 (float) | 百分比跳转 |
| VOLUME | 0-100 (int) | 设置音量 |
| MUTE | 1/0 | 静音/取消 |
| OPEN | 文件路径 | 打开视频文件 |
| 3D | off/sbs/tab/... | 3D 模式 |
| ASPECT | 无 | 循环宽高比 |
| POS | 无 | 查询位置（返回 POS:ms:totalMs） |
| PORT | 端口号 | 设置模块接收端口 |

## 3. 功能列表

### 已实现
- ✅ 通过 UDP 发送 PLAY/PAUSE/STOP 到所有 KODI
- ✅ 通过 UDP 发送 SEEK 到所有 KODI
- ✅ 通过 UDP 发送 VOLUME/MUTE 到所有 KODI
- ✅ 接收 ACK 反馈
- ✅ PORT 命令告诉插件模块的接收端口
- ✅ commands: Play All / Pause All / Stop All / Seek All / Set Volume All / Mute All / Set 3D Mode All / Set Aspect All

### 待实现
- ⬜ 漂移监控（HTTP 查询位置，对比漂移）
- ⬜ ReSync（强制校准所有 KODI 位置）
- ⬜ 自动同步（定时校准）
- ⬜ 多声道音量独立控制
- ⬜ Drift 显示面板

## 4. 配置参数

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| Sync Enabled | Boolean | true | 启用同步 |
| Receive Port | Integer | 9528 | 接收 ACK 的本地端口 |
| Secondaries | String (multiline) | - | KODI IP 列表 |
| HTTP User | String | kodi | HTTP 认证用户 |
| HTTP Pass | String | tocentek | HTTP 认证密码 |
| Local Port (UDP Input) | Integer | 10000 | UDP 接收端口（需与 Receive Port 一致） |

## 5. 安装前置条件

每台 KODI 上需要安装 `service.chataigne-sync` 插件：
- UDP 监听端口：9527
- 接收文本协议命令
- 执行 JSON-RPC 到本地 KODI
- 发送 ACK 回模块指定端口

## 6. 使用流程

1. 加载 KODI Sync 模块
2. 在 Status → Secondaries 填入所有 KODI IP
3. 设置 Receive Port = 9528
4. 设置 Parameters → Input → Local Port = 9528（与上一步相同）
5. 点击 Play All 测试
6. 查看 ACK 确认所有 KODI 已收到

## 7. 已知问题

- `local.sendTo` 单播到每个 IP（不支持广播因为 SO_BROADCAST 问题）
- ACK 接收依赖 `hasInput=true` 和正确配置的 Local Port
- `dataReceived` 回调中 data 参数直接是字符串（无需 `.toString()`）
